3 вида контрагента:1) Юридическое лицо2) Индивидуальный предприниматель3) Физическое лицоЮр лицоТестовые случаи: - Обязательные поля заполнены правильно → создание успешно.- Некорректные или пустые обязательные поля → сообщение об ошибке и отказ в создании.- Дублирование данных (например, по ИНН или названию) → предупреждение или отказ в создании.- Создание с разными комбинациями данных (например, с пропущенными необязательными полями).- Права пользователя — проверка невозможности создать контрагента без соответствующих разрешений.- Проверка корректности обработки юридического адреса и фактического адреса- Автоматическая подстановка данных из регистрирующих документов- Проверка заполнения банковских реквизитовИпТестовые случаи:- Обязательные поля заполнены правильно → создание успешно.- Некорректные или пустые обязательные поля → сообщение об ошибке и отказ в создании.- Дублирование данных (например, по ИНН или названию) → предупреждение или отказ в создании.- Создание с разными комбинациями данных (например, с пропущенными необязательными полями).- Права пользователя — проверка невозможности создать контрагента без соответствующих разрешений.- Проверка выбора вида деятельности- Проверка допустимых и недопустимых значения ОГРНИП- Проверка выбора разных налоговых режимов, отображение и обработку.Физ лицоТестовые случаи: - Обязательные поля заполнены правильно → создание успешно.- Некорректные или пустые обязательные поля → сообщение об ошибке и отказ в создании.- Дублирование данных (например, по ИНН или названию) → предупреждение или отказ в создании.- Создание с разными комбинациями данных (например, с пропущенными необязательными полями).- Права пользователя — проверка невозможности создать контрагента без соответствующих разрешений.- Проверка ввода паспортных данных- Проверка учета дополнительных контактов (телефон, почта)
